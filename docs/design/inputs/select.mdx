## Behavior
<div className="mt-lg mb-sm">

### Focus State
The border will transition to blue-bright when in focus, and the chevron will flip, revealing the options.
</div>
<Container Caption="Click / Tap to see focus state" className="flex-column pa-sm">
    <div style={{zIndex:5}} className="w-100"><SelectExample/></div>
</Container>

<div className="mt-lg mb-sm">

### Error State
When in an error state, the border will transition to red with an error message, and the options will collapse.

</div>

    <div style={{zIndex:5}} className="w-100"><ErrorExample/></div>

## Usage guidelines 
<!-- Start of a guide -->
### Overflow Text
Size the select input according to the data you will be taking on. Since the input currently does not change in height 
input data will need to be designed around to avoid ugly overflow.

<Guide guidance="don't">
   <div style={{zIndex:5, width:'150px'}}><SelectExample/></div>
</Guide>

<br/> 


<!-- Start of a guide -->
### Clear Labels 
Labels in select inputs are essential as the options may 
read as different from each other without a label.
<Guide guidance="do">
   <div style={{zIndex:5, width:'250px'}}><PinkThings/></div>
</Guide>

## Specs

<div className="mt-lg mb-sm">

### Size
Select input has a fixed height of 48px, and the width takes 100% of its parent container. The height when options are revealed is 100% of the total amount of options.

</div>
<Container>
  <img width="60%"  height="auto" src='/img/design/input/select/specs.svg'/>
</Container>

<div className="mt-lg mb-sm">

### Spacing
Spacing of the select input is 8px padding top and bottom, 8px left, 16px right and 4px between the input and the 
error message.
</div>
